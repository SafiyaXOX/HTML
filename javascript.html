Learning JavaScript

Introduction to JavaScript

JavaScript is a high-level, multi-paradigm programming language primarily known for enabling interactive and dynamic content on web pages.It can update both HTML and CSS. Additionally, it can calculate, manipulate, and validate data.

Main Purposes

  1. Makes websites more interactive
    • Responds to user actions (clicks, typing, scrolling)
    • Updates content without reloading the page.
    • Creates interactive elements like buttons, forms, and animations.
  2. Manipulates web pages
    • Changes HTML content dynamically
    • Modifies CSS styles
    • Shows/ hides elements
    • Updates text, images, and structure
  3. Handles user input
    • Validates forms before submission
    • Processes button clicks
    • Responds to keyboard and mouse events
  4. Communicates with servers
    • Sends and receives data (AJAX/fetch)
    • Updates pages with new information
    • Works with APIs
  5. Creates dynamic content
    • Builds interactive games
    • Creates animaions and transitions
    • Builds single-page applications (SPAs)

How to Run JavaScript Code

JavaScript can be added within the HTML file or it can be externally linked into your file using the <script> tags. 

<!-- This is adding script within your HTML Document -->
<!DOCTYPE>
<html lang ="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
</head>
<body>
    <script>
        console.log ("Hello World");
        // Insert JavScript Here
    </script>
    <!-- '//' writes comments in JavaScript -->
</body>
</html>


<!-- This is linking external script to your HTML Document -->
<!DOCTYPE>
<html lang ="en">
    <head>
        <meta charset="UTF-8">
        <title>Document</title>
    </head>
    <body>
        <script src ="script.js"></script>
        <!-- External Link to JavaScript file script.js -->
    </body>
    </html>

Save and open your file in a web browser. Then, open the browser's console:

  1. Right-click on the blank webpage.
  2. Click on "Inspect" or "Inspect Element" to open the Developer Tools.
  3. Find and select the "Console" tab, where you should see the output of the console.log statement.
console.log () is the command that prints something to the developer console in the browser.

Variables and Operators

Variables

Variables are the building blocks of any program. You can think of variables as "storage containers" for data in your code.

You can declare variables using the let keyword. 

let firstName = "John";
let lastName = "Doe";
                
console.log (firstName);
console.log (lastName);
You can also re-assign variables: 
let age = 11;
console.log(age); // output 11

age = 54;

console.log(age); // what will be the output now?
Notice the lack of let on line 4 - you don't need it since the variable has already been declared earlier and we are just re-assigning it here.

If you don't want to re-assign a variable, we can give it the const keyword. 
const pi = 3.14;
pi = 10;

console.log (pi); // What will be the output?
You may think that the output will be 3.14, however, an error will be thrown instead.

Errors are incredibly helpfful at telling us what is wrong with out code, and exactly where the issue is. Without it, it would be frustrating to try to find what is wrong.

Summary:
  • let, which can be re-assigned.
  • const, which can't be re-assigned and will throw an error if you try.
There is a third way, var, which was the original way that variables were declared in JavaScript. Var is similar to let; in that variables assigned this way can be reassigned, but it has other quirks that were cleared up when the language introduced let and const. By and large, it is not used anymore.

However, you will likely come across code which uses var at some point, so it is useful to know that it exists.

Numbers

Numbers are the building blocks of programming logic. It is hard to think of any useful programming tasks that doesn't involve at least a little basic math, so knowing how numbers work is obviously quite important.

JavaScript follows the standard mathematical Order of Operations (often remembered by acronyms like PEMDAS or BODMAS). This means:

  1. Parentheses (or Brackets) are evaluated first.
  2. Multiplication and Division are done next, from left to right.
  3. Addition and Subtraction are done last, from left to right.
